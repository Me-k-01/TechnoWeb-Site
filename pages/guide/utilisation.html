Utilisation général des fonctionnalitées d'Atom

Les projets

Vous pouvez ouvrir dans Atom un dossier de projet simplement en glissant le dossier dans Atom. Vous pouvez également ouvrir le fichier en allant dans File > Open File et en sélectionnant votre fichier pour ceux qui aiment se compliquer la vie. Atom met également à votre disposition, la possibilité de rouvrir des projets qui ont déjà été ouverts précédemment, dans File > Reopen Project.

Les raccourcis

Lorsque l'on se met à se servir sérieusement d'un logiciel, il est toujours bon de connaitre quelques-uns de ses raccourcis fonctionnels pour améliorer sa productivité. C'est pourquoi dans cette section, nous traiterons quelques raccourcis de base qui pourrait vous servir pour être plus efficace dans l'éditeur.

  • Sauvegarder: "Ctrl" + "s"

    Pour sauvegarder le fichier.

  • Rechercher: "Ctrl" + "f"

    Pour rechercher un mot / une phrase dans le fichier courant.

  • La palette de commande: "Ctrl" + "Shift" + "p"

    Pour trouver n'importe quelle fonctionnalité présente dans Atom, même ceux de package que vous avez installé.

  • Fuzzy File Finder: "Ctrl" + "t"

    Pour rechercher rapidement un fichier dans le/les dossier(s) ouverts.

  • Mettre en commentaire: "Ctrl" + "/" (celui du pavé numérique)

    Permet de mettre ou de retirer en commentaire la ligne ou toute la sélection.

  • Dupliquer: "Ctrl" + "Shift" + "d"

    Permet de dupliquer la ligne/sélection.

  • Les curseurs multiple: "Ctrl" + Clique Gauche

    Pour ajouter un nouveau curseur dans l'éditeur. Tous les curseurs entreront les saisies claviers en même temps.

  • Nouvelle onglets: "Ctrl" + "n"

    Pour ouvrir un nouveau fichier dans un nouvel onglet

  • Nouvelle fenetre: "Ctrl" + "Shift" + "n"

    Ouvrir une nouvelle fenêtre vierge

Vous pouvez bien entendu modifier les raccourcis claviers à tout moment: D'abord allez dans la rubrique "Shortcut" des paramètres, puis cliquer sur l'icône de bloque note, pour copier le code. Une cela fait, il ne vous suffira plus que de cliquer sur "your keymap file". Une fois la fenêtre ouverte vous pouvez coller ce que vous avez copié, et modifier à votre guise la combinaison de touches demander pour accéder à cette fonction.

Les thèmes

Vous pouvez changer de thèmes en allant dans la rubrique thème des paramètres. Vous y trouverez quelques thèmes préinstallés. Mais s'ils ne sont pas à votre goût, vous pouvez toujours en télécharger depuis l'interface d'installation juste en bas.

Vous pouvez aussi personnaliser le look de votre éditeur vous-même en cliquant sur "your stylesheet". Le language est similaire au Css. plus d'information

L'intégration Git et GitHub

Atom étant créé par GitHub, il propose de gérer lui-même les modifications apportées à votre projet. Vous n'avez en effet plus besoin d'ouvrir le terminal de commande pour tapper les fameux "git push" et "git commit", l'intégration Git prend en charge tout ceci.

Pour ce faire, il faudra d'abord vous connecter une première fois avec votre compte GitHub sur Atom (si vous n'avez pas de compte, vous pouvez en créer un sur https://github.com/), puis une fois que cela est réalisé, vous pourrez utiliser pleinement l'intégration Git dans vos prochains projets.

Avant de pouvoir utiliser pleinement cette intégration, vous aurez surement compris qu'il faut d'abord ouvrir le projet dans atom. Pour cela, rien de plus simple, il vous suffit d'ouvrir le menu git clone (accessible en ouvrant la palette de commande et en tapant "git clone"). Une fois dans le menu de git clone, il vous faut entrer le lien de du dépôt GitHub du projet (ex: https://github.com/VotrePseudo/VotreProjet), vous pouvez aussi lui préciser l'endroit sur lequel vous voulez qu'il enregistre le projet. Par défaut, il le mettra dans un fichier "github" qu'il créera pour vous, dans votre répertoire personnel.

Lorsque vous avez ouvert dans Atom le répertoire d'un de vos projets, vous pouvez ouvrir la fenêtre Git en appuyant sur le button éponyme, en bas à droite de l'éditeur.

Une fois ouvert, cette intégration vous donnera la possibilité de visualiser et d'ajouter tout changement direct du projet sur le projet (suppression de doc...). Lorsque vous êtes content de votre travail, vous pouvez appuyer sur "Stage All" qui va "commit" tous les changements. (vous pourriez les validés un à un manuellement, mais cela deviendrait vite redondant). Ces changements sont bien entendus encore réversibles à ce stade-ci (ouf).

Le bouton "Commit to master" vous permettra d'effectuer ces modifications sur la branche principale de votre projet (soit la branche master normalement). Ce qui signifie effectivement, (pour les plus perspicaces d'entre vous) que l'on peut séparer un dépôt en plusieurs branches. Et même fusionner des branches entre elles.

Pourquoi me diriez-vous?

Et bien cela prend sens pour les personnes travaillant sur de gros projets, comme pour certaines applications. De cette manière un développeur peut corriger de son côté un certain bug, sans affecter toute la branche principale qui pourrait servir de distribution pour cette application hypothétique. Cette fonctionnalité peut aussi s'avérer très pratique lorsque l'on travaille en groupe sur le même projet.

Une fois que vous êtes prêt à valider vos changements, donc après avoir appuyé sur "Commit to master", vous pouvez cliquer sur "push" en bas à droite, qui va effectuer les changements sur le dépôt GitHub.

Vous pouvez également "fetch", toujours au même endroit que "push", pour synchroniser votre projet au dépôt GitHub, si jamais il y a eu des modifications.
